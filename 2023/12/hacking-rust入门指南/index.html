Hacking Rust：入门指南

Hello world!

用 Rust 打印 hello world!：

1
2
3

fn main() {
    println!("hello world!");
}

关于上述代码的几点解释：

  • main 函数是 Rust 程序的入口。

  • println! 是 Rust 语言的宏。如果是函数的话应该去掉 !，这也是函数和宏的区别。

  • 大部分 Rust 代码都以分号结尾。

编译和运行

Rust 是预编译静态类型语言，也就是说，编译和运行是独立的。我们可以编译后将可执行程序发送给别人直接运行而无需安装环境。

1
2
3
4
5

# 编译
rustc main.rs

# 运行
./main

Cargo： 构建系统和包管理器

Cargo 用来构建代码、下载依赖并编译这些库。

cargo new hello_cargo 创建一个项目。目录结构如下：

.
├── .git
│  ├── config
│  ├── description
│  ├── HEAD
│  ├── hooks
│  ├── info
│  ├── objects
│  └── refs
├── .gitignore
├── Cargo.toml
└── src
   └── main.rs

其中 Cargo.toml 文件内容如下：

1
2
3
4
5
6

[package] # 一个片段（section），表明下面的语句用来配置一个包
name = "hello_cargo" # 项目名称 
version = "0.1.0" # 项目版本
edition = "2021" # 使用的 Rust 版本

[dependencies] # 项目依赖，在 Rust 中，代码包被称为 crates

  • cargo build 构建项目。运行后会：

    • 自动生成 Cargo.lock 文件，记录项目依赖的实际版本。

    • 默认使用调试构建（debug build），因此生成的文件会放在 target/debug 目录下。

  • cargo run 同时完成编译和运行的步骤。

  • cargo check 检查代码是否可以编译而不产生可执行文件，执行速度要比 cargo run 快很多。

  • cargo build --release 优化编译项目，这会在 target/release 文件夹生成可执行文件，可以让代码运行更快，但需要消耗更多的编译时间。
